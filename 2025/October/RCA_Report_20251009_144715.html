📋 Summary of the Issue

The customer, Astuto, was experiencing an issue where Grafana logs were not visible in their production environment. The expected behavior was for the logs to be collected and displayed by Grafana, but this was not happening. The issue was traced to the promtail DaemonSet not running on ARM-based nodes where the Dagster pods were deployed. The promtail DaemonSet was configured to run only on non-ARM nodes, and the ARM nodes had a taint that prevented scheduling without the appropriate toleration. This misconfiguration led to the logs not being collected from the ARM nodes, impacting the visibility of logs in Grafana.

✅ Steps to Resolution

1. **Rauneet Verma** recommended updating the loki_gcs log_collector resource to include the necessary tolerations in the promtail section. This would allow the promtail DaemonSet to schedule on ARM-based nodes and collect logs from the Dagster services:
📝 YAML
"promtail": {
       "values": {

         "tolerations": [

           {

             "key": "application",

             "operator": "Equal",

             "value": "onyx",

             "effect": "NoSchedule"

           },

           {

               "key": "kubernetes.io/arch",

               "operator": "Equal",

               "value": "arm64",

               "effect": "NoSchedule"

           }

         ]

       }

     },

2. The customer was advised to implement this configuration change to enable promtail to run on ARM nodes, thus resolving the log visibility issue in Grafana. The status of the ticket indicates that this was a customer-side fix, suggesting that the customer was responsible for applying the changes.

🎯 Root Cause Analysis

The root cause of the issue was the absence of a necessary toleration in the promtail DaemonSet configuration, which prevented it from running on ARM-based nodes where the Dagster pods were deployed. The ARM nodes had a taint kubernetes.io/arch=arm64:NoSchedule, and the promtail DaemonSet was not configured to tolerate this taint. As a result, promtail could not collect logs from these nodes, leading to the logs not being visible in Grafana. The issue was identified through a detailed examination of the node configurations and the promtail DaemonSet setup. The customer was advised to update the configuration to include the required toleration, allowing promtail to run on ARM nodes and collect the necessary logs.

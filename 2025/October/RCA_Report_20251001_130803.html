📋 Summary of the Issue

The customer, MoveInSync, was attempting to enable the ngx_brotli module in their development and staging environments using the NGINX Ingress Controller. The expectation was that Brotli compression would be applied to various file types, including JavaScript files. However, Brotli was only being applied to SVG files, while JavaScript files continued to use gzip compression. This discrepancy was due to the NGINX Ingress Controller's default behavior of prioritizing gzip over Brotli when both are enabled. The environment involved Kubernetes clusters with specific configurations for the ingress controller.

🎯 Root Cause Analysis

The likely root cause of the issue was the NGINX Ingress Controller's default behavior of prioritizing gzip compression over Brotli when both are enabled. This behavior is due to the controller's inability to configure the precedence of compression techniques, as confirmed by a related GitHub issue. Additionally, the backend service's NGINX configuration had gzip enabled, which resulted in content being compressed with gzip before reaching the ingress, preventing Brotli from being applied. Disabling gzip in the backend and adjusting the ingress configuration to prioritize Brotli were necessary steps to resolve the issue.
